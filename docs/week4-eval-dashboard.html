Tutor Safety Contract

GenAcademy Coach is a personalized, adaptive, grounded AI tutor over the Gen Academy course corpus. The contract is: if it can cite the course material, it teaches. If retrieval confidence is low, it refuses and escalates. It must never answer from model priors.

Evaluation Setup & Scope

Evaluated against dataset version 2026-06-24-plan1 (frozen 40-case golden set: 16 happy, 9 edge, 5 known_failure, 10 adversarial). Evaluators combine code-based checks (deterministic Python metrics), trajectory eval (path-level next action and citation verification), and human review. LLM-as-judge is deferred to future offline audits.

Evaluator Types

Code-based checks

Deterministic Python checks for labeled outcomes: task completion, citations, tools, retrieval, refusal, latency, tokens, cost, and privacy guards.

Trajectory eval

Path-level scoring: what the agent retrieved, which tools it used, what action it chose, whether it cited evidence, and whether it advanced, re-explained, or refused in the right place.

Human review

Reviewer inspection of the report, dashboard, and sampled trace evidence for pedagogical quality, caveats, and public-safe redaction.

Observability: Local eval runs emit redacted JSON diagnostics. LangSmith is used for owner-approved private trace review only; no private traces or urls are exposed.

Product Behavior Changed, Not The Scorer

The 40-case golden set, labels, and deterministic scorer stayed fixed. The changes were inside the tutor: how it advances, cites, retries, caps repeated calls, and reports latency without exposing private text.

01

Correct + grounded answers advance

Problem
Correct learner answers could still trigger another drill or re-explanation.
Product change
When Python grading says the answer is correct and grounded, the session advances.
Eval proof
Trajectory scoring checks the selected action, not only final prose.
02

Checks prefer stronger citation spans

Problem
Retrieval found evidence, but the agent sometimes used a weaker note or transcript span.
Product change
Check generation prefers slide evidence first, handouts second, then another citable span.
Eval proof
Citation F1 measures whether the chosen citation matches the labeled support.
03

Malformed model output gets one retry

Problem
A missing or invalid structured response could derail a teach turn.
Product change
The product retries once and counts the extra tokens instead of hiding the cost.
Eval proof
Token and latency metrics keep the retry visible.
04

Repeated calls are capped or reused

Problem
Retrieval, check generation, or escalation could repeat inside a turn.
Product change
The loop caps or reuses repeated retrieval/check/escalation calls.
Eval proof
Tool-call counts and latency attribution show whether the loop stayed bounded.
05

Latency attribution stays redacted

Problem
The reviewer needs cost and speed evidence without seeing private learner or corpus text.
Product change
The dashboard reports aggregate latency, tokens, and tool shares only.
Eval proof
The public snapshot guard blocks private fields and URLs.

Why Eval Mattered: Rejected Levers

A broad citation fallback was proposed to anchor final answers and improve citation F1. However, evaluation showed it caused a regression: citation F1 fell from 0.444 to 0.400 and task completion dropped by 5.2 pp. The change was rejected and not shipped. The eval was our debugger.

Production Monitoring

Batch eval catches regressions before release; these are the live signals we would alert on for the grounded teach loop.

SignalMetricAlert thresholdWhy
Quality drift Citation F1 & task-completion (7-day rolling) Alert if either drops > 10% vs trailing baseline Catches retrieval/citation regressions before learners feel them.
Cost spike p95 tokens & cost per turn (24h) Alert if > 25% over budget Usually signals a tool-loop or retrieval blow-up.
Latency regression Turn p95 latency Alert if > 10s SLA on > 5% of runs Protects the interactive teach loop.
Guardrail trips Refusal + escalation rate Alert if > 2x trailing baseline Over-conservative refusal is our dominant failure mode — watch it live.

Grader Evolution & Roadmap

Next steps for tutor evolution, strictly respecting the Grader Evolution risk-cap: future checks remain versioned separately and evaluated on the frozen golden set.

Semantic Grading

Move from literal keyword checking to deterministic synonym groups and optional embedding similarity. Grader changes will be versioned separately and re-evaluated on the same golden set, never retrofitted.

Turn-2 Copy Polish

Improve the conversational flow when the session advances so the transition feels natural.

Targeted False-Refusals

Calibrate thresholding to address over-conservative refusals in teachable cases (our main task failure mode).
