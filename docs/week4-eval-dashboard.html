Evaluator Types

This dashboard combines code-based scoring, trajectory checks, and human review. LLM-as-judge is intentionally deferred to future offline audit work, not the v1 pass/fail scorer.

Code-based checks

Deterministic Python checks for labeled outcomes: task completion, citations, tools, retrieval, refusal, latency, tokens, cost, and privacy guards.

Trajectory eval

Path-level scoring: what the agent retrieved, which tools it used, what action it chose, whether it cited evidence, and whether it advanced, re-explained, or refused in the right place.

Human review

Reviewer inspection of the report, dashboard, and sampled trace evidence for pedagogical quality, caveats, and public-safe redaction.

Production Monitoring

Batch eval catches regressions before release; these are the live signals we would alert on for the grounded teach loop.

SignalMetricAlert thresholdWhy
Quality drift Citation F1 & task-completion (7-day rolling) Alert if either drops > 10% vs trailing baseline Catches retrieval/citation regressions before learners feel them.
Cost spike p95 tokens & cost per turn (24h) Alert if > 25% over budget Usually signals a tool-loop or retrieval blow-up.
Latency regression Turn p95 latency Alert if > 10s SLA on > 5% of runs Protects the interactive teach loop.
Guardrail trips Refusal + escalation rate Alert if > 2x trailing baseline Over-conservative refusal is our dominant failure mode — watch it live.
Tool failure Per-tool error rate: retrieve / generate_check / grade (1h) Alert if any single tool > 5% Flags external-dependency outage or structured-output regression.
